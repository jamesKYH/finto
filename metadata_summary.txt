# 외국환거래법 메타데이터 구조 및 검색 방법 분석

## 1. 메타데이터 구조 분석

### 메타데이터 기본 구조
- 모든 항목(294개)은 다음과 같은 기본 메타데이터를 포함함:
  - law_name: 외국환거래법
  - effective_date: 법령 시행일 (2021. 9. 16.)
  - publication_info: 공포 정보 (법률 제18244호, 2021. 6. 15., 일부개정)
  - ministry: 소관부처 (기획재정부)
  - chapter_no: 장 번호 (예: 제1장)
  - chapter_title: 장 제목 (예: 총칙)
  - section_type: 섹션 유형 (조문, 기타 등)

### 계층적 메타데이터 구조
- 항목은 '장-조-항' 구조로 계층화되어 있음:
  1. 장(chapter): 모든 항목(100%)이 장 정보를 포함 (총 6개 장)
  2. 조(article): 81%의 항목이 조문 정보를 포함 (총 36개 조문)
  3. 항(item): 53.4%의 항목이 항 정보를 포함 (① ~ ⑮ 표기법 사용)

### ID 체계
- ID는 대부분 '장번호-조번호' 또는 '장번호-조번호-항번호' 형태로 구성됨:
  - 2단계(장-조): 예) 1-1, 1-2, 2-10 등
  - 3단계(장-조-항): 예) 1-2-①, 1-3-①, 2-10-③ 등
- ID 값은 메타데이터의 장, 조, 항 번호와 일치하는 패턴을 보임

## 2. 검색 및 조회 방법

### 메타데이터 기반 검색
1. 장 번호로 검색: 
   - 메타데이터의 chapter_no 필드 사용 (예: "제1장")
   - 제1장에는 총 25개 항목이 존재

2. 조 번호로 검색:
   - 메타데이터의 article_no 필드 사용 (예: "제1조")
   - 제1조에는 총 15개 항목이 존재 (다양한 장에 분포)

3. 장-조 조합 검색:
   - chapter_no와 article_no 필드 조합 사용
   - 제1장 제1조에는 1개 항목만 존재

4. 장-조-항 조합 검색:
   - chapter_no, article_no, item_no 필드 조합 사용
   - 제1장 제3조 ① 항에는 2개 항목이 존재

### 개선 방안
1. 메타데이터에 현재 포함된 장(chapter_no)과 조(article_no) 정보는 검색에 매우 유용함
2. 항(item_no) 정보도 잘 구조화되어 있어 상세 검색 가능
3. 개선점:
   - 일부 중복 데이터 존재 (예: 제1장 제3조 ① 항이 2개 존재)
   - 검색 시 메타데이터와 ID 구조를 모두 활용하면 더 정확한 검색 가능
   - 벡터 검색과 메타데이터 필터링을 조합하여 사용하면 효과적임

## 3. 제3장(외국환평형기금) 데이터 특성
- 제3장에는 총 17개 항목이 포함됨
- 제3장에는 제13조와 제14조의 두 가지 조문만 존재
- 메타데이터 구조는 다른 장들과 동일한 패턴 유지

## 4. 결론
- 현재 메타데이터 구조는 장-조-항 체계를 잘 반영하고 있어 검색에 효과적임
- ID 체계도 이러한 구조를 일관되게 반영하여 검색과 분류에 도움이 됨
- 제1장 제1조 검색 결과가 정확하게 나오는 것을 확인함
- 메타데이터 필드를 사용한 세부 검색이 가능하여 사용자 요구에 맞는 정확한 법령 정보 제공 가능
